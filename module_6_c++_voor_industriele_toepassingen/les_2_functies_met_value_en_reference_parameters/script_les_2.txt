Welkom bij les 2 van de cursus C++ voor industriele toepassingen,
onderdeel van de Wiztech lessenserie.
Mijn naam is Jacques de Hooge.

Functies zijn een manier om een programma op te delen in deeltaken, die deeltaken weer in subtaken en zo verder.
Die manier van opsplitsen van je programma wordt wel function structured programming genoemd,
en was state of the art in de jaren 60 en 70 van de vorige eeuw.
Verwar function structured programming niet met functional programming, dat is iets heel anders en van recenter datum.

Function structured programming was in de taal C the way to go.
Ik weet nog dat ik voor mijn eerste klant een C programma moest aanpassen dat uit zo'n 600 losse functies bestond.
Het heeft weken geduurd voor ik doorhad hoe al die losse puzzelstukjes samen een plaatje vormden.
Ik had duidelijk behoefte aan een extra organisatielaag.

Nu valt dat ook in C wel te regelen, maar de komst van C++ was voor mij een feest van herkenning.
Daar zat die organisatielaag namelijk van nature in:functies waren gegroepeerd rond objecten.
Het was of alle puzzelstukjes al op kleur gesorteerd waren en ook grote puzzels waren opeeens geen probleem meer.
Ik voelde me als een kind in de snoepwinkel, daarover meer in de volgende lessen.
Maar eerst functies, dus.
